## O que é e por que o Markdown é importante?

**Markdown** é uma linguagem de marcação leve, criada para facilitar a escrita e leitura de textos na web. Em vez de usar códigos HTML complexos ou menus de editores visuais (como o Word). É utilizado símbolos simples no teclado para formatar o conteúdo.

É uma função importante devido sua simplicidade e foco. Ele pode ser aberto e lido praticamente em qualquer editor, sem depender de softwares específicos, pois é baseado em puro texto. 

Isso faz com que seja um padrão ouro para documentação de projetos em TI, sendo amplamente utilizado para criar arquivos README.md.

## Estruturas e Sintaxes do Markdown

- **Enfatizar Textos**:
Utilize ** para negrito e * para itálico. É a base para destacar termos importantes em documentações técnicas.

- **Declarar Cabeçalhos**:
Use o caractere # no início da linha para criar títulos. A quantidade de # define a hierarquia (de H1 a H6).

- **Criar Listas**:
Para listas não ordenadas, use hifens - ou asteriscos *. Para listas numeradas, utilize números seguidos de ponto (ex: 1. ).

- **Inserir Imagens**:
A sintaxe segue o padrão "!+[texto-da-imagem]+(url-da-imagem)". Isso permite incluir prints de tela, como o arquivo image.png, para ilustrar seu código.

- **Criar Texto**:
Basta digitar o conteúdo normalmente. Para separar parágrafos, deixe uma linha em branco entre eles.

- **Preencher lacunas com HTML inline**:
O Markdown permite o uso de tags HTML (como <br> para quebras de linha) quando a sintaxe padrão não atende a uma necessidade específica de design.

- **Trabalhar com código**
Envolva códigos curtos em crases (`). Para blocos maiores, como scripts em PHP ou C#, use três crases (```) seguidas do nome da linguagem para ativar o realce de sintaxe.

- **Vincular problemas e solicitações de pulls**
O GitHub converte automaticamente o símbolo # seguido do número (ex: #3) em um link para a respectiva Issue ou Pull Request dentro do repositório.

- **Mencionar usuários ou equipes**
Use o símbolo @ seguido do nome de usuário (ex: @leticiakonno) para marcar e notificar colaboradores sobre alterações ou revisões necessárias.

- **Listar Tarefas**
Crie checklists de progresso usando - [ ] para itens pendentes e - [x] para itens concluídos.

- **Comandos de Barra**
Ao digitar / no editor do GitHub, um menu de atalhos é exibido para inserir rapidamente elementos como tabelas, listas ou templates de resposta.
